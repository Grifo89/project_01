Design Language

A refined, utilitarian design system built for local-first productivity. Precise, confident, and respectful of user focus. Every decision is mobile-first and adaptive to dark environments.

Preact + Vite Tailwind v4 PWA / Local-First Mobile-First

⚠ Reduced Motion

Always wrap non-essential animations in @media (prefers-reduced-motion: no-preference). State transitions (loading spinners, progress fills) are exempt — they communicate function.

Do's & Don'ts

Common mistakes to avoid when extending the design system.

✓ Do
In Progress

Use semantic badge variants that match the meaning: primary for active, success for done, warning for review.

✗ Don't
In Progress

Don't invent one-off badge colors. Stick to the 6 semantic variants.

✓ Do

Use uppercase overline labels (10px / 800 / tracked). They recede and let content breathe.

✗ Don't

Don't use regular-weight large labels. They compete with the input value.

✓ Do

Tasks Done

18

Use border + shadow together on cards.

✗ Don't

Tasks Done

18

Don't use shadow alone — invisible in dark mode without a border.

✓ Do

Progress fill is always emerald green (#10b981). It means completion.

✗ Don't

Don't use brand blue for progress. Blue means "active/clickable", not "done".

CSS Variable Rules

/* ✓ Always use tokens — adapts to dark mode automatically */ color: var(--text-primary); background: var(--bg-surface); border: 1px solid var(--border-subtle); /* ✗ Never hardcode colors in components */ color: #101622; /* breaks in dark mode */ background: #FFFFFF; /* breaks in dark mode */

Accessibility Checklist

RuleRequirement
Color contrast — text on bg≥ 4.5:1 for body text, ≥ 3:1 for large text (WCAG AA)
Focus ringsAll interactive elements must have visible focus ring: box-shadow: 0 0 0 3px var(--primary-soft)
Touch targets (mobile)Minimum 44×44px. Never smaller.
MotionAll animations wrapped in prefers-reduced-motion: no-preference
Semantic HTMLUse <button> not <div onclick>. Use <nav>, <main>, <section>.
Icons without textMust have aria-label or visually-hidden text
Form inputsEvery input must have an associated <label> (visible or sr-only)
